## Что сделано за сегодняшнюю сессию (по порядку)

1. **Advisor v1** (`ADVISOR_DESIGN.md` пункты 1-3, 5-6) — rule-engine с приоритетом: видимость → рынок против тебя → долг+простой → узкий спред+стух → широкий спред+свежо → нейтрально. Каждый совет содержит конкретную цену (`baseline_price`/`aggressive_price`). Эндпоинт `/api/advice`, карточка "Совет сейчас".

2. **Баг видимости (повторный)**: заявка без общего способа оплаты "Bank Transfer" (`BANK`) невидима для большинства покупателей, даже если ранг без фильтра выглядит хорошо. Чинили один раз на BUY-заявке в прошлую сессию, сегодня поймали тот же баг на SELL. **Проверять способы оплаты на каждой новой заявке.**

3. **Outlier-resistant pricing**: один аномальный ордер мог утянуть `best_buy_price`/`best_sell_price` и сломать расчёт спреда (был случай -39 IDR спред). Добавлены `robust_buy_price`/`robust_sell_price` (медиана топ-3) — используются в спреде, рекомендуемых ценах, advisor'е. Сырые best-цены оставлены только для информационных лейблов.

4. **Баг хронологии в P&L**: старый FIFO мог матчить продажу против покупки, случившейся *позже* неё по времени (физически невозможно) — раздувало/искажало "реализованный P&L". Переписан `pnl_tracker.summary()`: честный time-respecting FIFO, новое поле `unmatched_sell_usdt` (часть продажи без известной себестоимости — профинансирована непротрекенным балансом), детект торговой сессии по разрыву >3ч.

5. **Карточка "$/час"** (`/api/earnings`) — реализованный P&L сессии + unrealized-оценка открытой позиции по текущей цене, делённое на часы. Принцип "думать в $/час, не $/день" из ADVISOR_DESIGN.md пункт 5.

6. **Поле "Доступно в банке"** (`/api/bank-balance`, GET/POST) — ручной ввод, ограничивает рекомендацию лимита BUY-заявки (банк не виден через API в принципе). Сравнение с реальным `max_limit` заявки (✓/⚠) на дашборде.

7. **Авто-реконнект** `market_watcher` при разрыве CDP-сессии (`_reconnect_after_error()` + `asyncio.Lock`) — чинит сегодняшний инцидент (Opera-вкладка умерла, бот 4.5 часа молча отдавал замороженные данные).

8. **История стакана + скорость рынка** (ADVISOR_DESIGN.md пункт 4, который сам документ считал необязательным): `price_trend_idr_per_min()` — 15-минутная история, signed-тренд. При быстром рынке (>3 IDR/мин) пороги "жди"/"скинь по застою" урезаются вдвое.

9. **Правило "рынок против тебя"** — приоритетнее долга и спреда: устойчивый тренд (>2 IDR/мин за ~10 мин) против активной стороны → немедленно "Скинь", независимо от возраста долга. Найдено после реального убыточного цикла сегодня (купил 17945, рынок падал, продал 17939 через 38 минут, -Rp 2337.66) — синтетический тест подтвердил, что новое правило поймало бы это раньше.

10. **Дашборд пересобран целиком** через `/frontend-design` — эстетика "приборная панель/терминал" (Rajdhani + IBM Plex Mono, угловые HUD-скобки, цвета amber/cyan/red). Сигнал "рынок против тебя" — полноэкранный пульсирующий баннер, невозможно пропустить. `advice.side` стал единственным источником истины для активной стороны (раньше дублировалось на фронте). Журнал сделок — со скроллом.

11. **Двуязычность RU/EN**: `advisor.py` строит текст из `STRINGS[lang]`, возвращает стабильный `state` (push/trend/debt/tight/wide/neutral) — фронт не парсит текст для цвета карточки. `main.py`: `/api/advice?lang=ru|en`, `market_direction` — языконезависимый код (up/down/flat/unknown). Фронт: словарь `I18N`, переключатель в шапке, сохранение в `localStorage`. Переключение языка форсирует немедленный re-fetch всех панелей (иначе часть текста отставала до 90 сек).

12. **"Рабочая сессия"** — новая концепция, отдельная от торговой сессии в P&L (та по разрыву >3ч между сделками). Каждый новый ввод в поле "Банк, IDR" фиксирует начало новой рабочей сессии (`work_sessions` в `main.py`, in-memory). Старая сессия "закрывается" с честным P&L (IDR) и числом циклов за её время. Эндпоинт `/api/work-sessions`, карточка на дашборде.

## План переноса на VPS3 (следующий крупный шаг)

VPS3 (Малайзия) не нуждается в VPN для доступа к Binance (проверено напрямую), ресурсов достаточно (2 CPU, 7.8GB RAM, занято 1.9GB) несмотря на уже работающие live-боты (Boba1/Grogu1 straddle).

**Главный риск — не ресурсы, а доверие Binance к браузерной сессии**: перенос уже залогиненной Opera-сессии на датацентровый IP — новое устройство/локация для антифрод-системы (в отличие от API-ключа, который уже работает с VPS3 без проблем).

**Согласованный путь**: НЕ переносить саму сессию — Opera с логином остаётся на Mac, через SSH reverse-tunnel порта 9222 на VPS3 крутится сам backend (FastAPI/uvicorn) 24/7. Полный перенос сессии — отдельный шаг на будущее, не сейчас.

## Как поднять всё с нуля после перезагрузки Mac

```bash
# 1. Opera с remote debugging
/Applications/Opera.app/Contents/MacOS/Opera --remote-debugging-port=9222 \
  --user-data-dir="$HOME/Library/Application Support/com.operasoftware.Opera"
# открыть в ней p2p/c2c.binance.com, залогиниться, включить VPN

# 2. Backend
cd ~/Desktop/Check/backend
source venv/bin/activate
uvicorn main:app --host 0.0.0.0 --port 8088
```

Дашборд: `http://localhost:8088`. Ключи API — в `backend/.env` (не в git). После правок `pnl_tracker.py` синкать на VPS3: `scp backend/pnl_tracker.py root@187.127.114.34:/root/pnl_tracker.py`.
