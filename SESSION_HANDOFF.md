## Архитектура (актуальная)

- `market_watcher.py` — читает стакан через CDP с уже залогиненной вкладки Opera. `find_my_ad()` теперь возвращает `adv_no` (нужен для редактирования объявления).
- `advisor.py` — rule-engine, приоритет: `push` (видимость) → `trend` (рынок против) → `debt` (долг+простой) → `tight` (узкий спред+стух) → `wide` (широкий спред+свежо, явно ЖДАТЬ) → `neutral`. Не менялся сегодня, но теперь `neutral` тоже триггерит авто-репрайс (см. ниже).
- `pnl_tracker.py`/`pnl_remote.py` — на VPS3 по SSH. **Сегодня добавлено**: `_fifo_annotate()`/`annotated_cycles()` — каждая SELL-строка несёт `profit_idr` (реальная FIFO-прибыль), каждая открытая BUY-строка несёт `open_usdt` (остаток, ещё не продан). При правке `pnl_tracker.py` — синкать на VPS3 (`scp backend/pnl_tracker.py root@187.127.114.34:/root/pnl_tracker.py`), уже сделано в этой сессии, расхождения нет.
- **`ad_repricer.py` — НОВЫЙ файл.** `AdRepricer` класс, управляет реальной формой `/en/advEdit?code={advNo}` через Playwright (отдельная вкладка, не трогает пользовательские). См. подробности ниже.
- `main.py` — добавлены `/api/auto-reprice` (GET/POST), `/api/active-side` (GET/POST), фоновый `auto_reprice_loop()`, вынесена общая `_advice_inputs()` (раньше дублировалась в `/api/advice`).
- `static/index.html` — добавлены: чекбокс "Auto-reprice" + статус-строка, три кнопки "Auto/BUY/SELL" под карточкой совета, звук монеток (Web Audio API, без файлов), колонка "Прибыль" в журнале сделок.

## Что сделано за сегодняшнюю сессию (по порядку)

1. **Прибыль по строкам в журнале сделок.** `pnl_tracker._fifo_annotate()` — каждая SELL-строка получает `profit_idr` (реальная FIFO-прибыль этой конкретной продажи, не общий тотал), `unmatched_usdt` если часть продажи не на что списать. По просьбе пользователя расширено: каждая ещё не проданная BUY-строка получает `open_usdt`, и `/api/pnl` на лету считает `unrealized_profit_idr` по живой цене (это уже Mac-сторона, у VPS нет доступа к live price). Фронт красит `Rp X` (реализовано) или `~Rp X` (нереализовано, тильда).

2. **Звук "монеток".** Web Audio API, синтезируется на лету (двойной "cha-ching" + россыпь щелчков) — без внешних файлов. Триггер: рост `total_trades` между опросами `/api/pnl` (раз в 90с) — не реагирует на пропажу заявки из выдачи (это может быть просто рейтинг, не сделка). Браузер требует один клик по странице после загрузки, чтобы разрешить звук (политика автоплея) — разблокировка повешена на первый клик где угодно.

3. **Перестроена логика лимитов BUY-заявки** при пополнении банка до Rp 40,000,000 — рекомендовал min 1M / max 39M (банк минус буфер), пользователь применил вручную. Не автоматизировано на дашборде (как и раньше).

4. **Авто-перестановка цены — основная фича сегодня.**
   - Сначала пробовал голый `fetch()` к найденному через сниффинг трафика эндпоинту `bapi/c2c/v3/private/c2c/adv/update` — Binance вернул `"Please log in first"` даже на залогиненной странице. Приватные эндпоинты требуют доп. заголовков, которые ставит их собственный JS, не только cookies. **Вывод: голый API-replay не работает для приватных эндпоинтов**, нужна настоящая UI-автоматизация.
   - Нашёл реальный flow: `/en/advEdit?code={advNo}` → `input[name="rate"]` (поле цены) → кнопка "Post" (включается после `fill()`) → **модалка "Confirm to post"** (это пропустил в первой версии — код кликал только "Post" и репортил успех, хотя реальная заявка не отправлялась, цена просто зависала в открытой вкладке). Нашёл это благодаря пользователю, который смотрел на реальный экран и сказал "там кнопка ещё одна".
   - `AdRepricer.reprice()` теперь: fill → poll Post enabled (не fixed sleep — гонка) → click Post → wait "Confirm to post" modal → click confirm → **проверка по факту**: переход на `/en/myads`, чтение реальной цены из таблицы (НЕ поиск по рынку — он лагает, показывал старую цену ещё ~30-60с после реального изменения).
   - Защиты: kill switch выключен по умолчанию (`repricer.enabled=False`), max delta 50 IDR за раз, cooldown 120с на объявление, авто-отключение при намёке на 2FA/verification в тексте страницы, отдельная вкладка (не трогает пользовательскую), `finally`-блок закрывает вкладку/браузер даже при ошибке.
   - Триггер: `auto_reprice_loop()` (каждые 30с) — состояния `push/trend/debt/tight/neutral` (т.е. ВСЕ, кроме `wide` — там advisor сам говорит ждать), и только если `recommended_price` отличается от текущей цены заявки на ≥1 IDR.
   - **Подтверждено живьём 3 независимыми способами** (лог auto_reprice, `/api/orderbook` поиск, прямой заход на `/en/myads`) — все сошлись на одной цифре после нескольких реальных перестановок (17925→17918 первая попытка с багом откатилась некорректно, после фикса: 17927→17928→17936, всё verified).
   - Дашборд: чекбокс "Auto-reprice" под карточкой совета + статус-строка с последним результатом.

5. **Ручной override активной стороны.** Авто-логика (`debt_state` в `main.py`) определяет BUY/SELL по lifetime FIFO net-flow ("купил больше чем продал → у тебя излишек, продавай") — не знает про **сессионную цель** пользователя (например "ещё $765 нужно докупить"). Сегодня поймали баг живьём: бот переключился на SELL-совет, хотя BUY-заявка с доступным остатком всё ещё валидна и в топе рынка. Добавлен `/api/active-side` (GET/POST `{"override": "BUY"|"SELL"|null}`) — проверяется ПЕРВЫМ, до авто-логики, используется и в `/api/advice`, и в `auto_reprice_loop`. На дашборде — три кнопки Auto/BUY/SELL под карточкой совета. **Override не персистится — сбрасывается в `null` (auto) при каждом рестарте backend.**

6. **Стабильность инфраструктуры — повторяющаяся проблема за сессию:**
   - Backend падал минимум дважды без видимой причины (возможно, терминальная сессия закрывалась) — чинилось рестартом.
   - Opera теряла `--remote-debugging-port` флаг при падении/перезапуске (видимо, авто-восстановление вкладок системой не сохраняет флаги запуска) — чинилось `osascript -e 'tell application "Opera" to quit'` + ручной релонч с флагом.
   - **В конце сессии — Mac сам перезагрузился** (`uptime` показал `up 2 mins`), убив Opera и backend одновременно. Восстановлено вручную (см. инструкцию внизу).
   - **Вывод**: пока это всё держится на ручных рестартах через Claude. Если нужна реальная "работает как часы" надёжность — следующий шаг: `launchd`-агенты, которые держат и Opera (с флагом), и backend живыми, плюс перезапускают при падении автоматически.

- **Коммит в git** — всё сегодняшнее только локально.

## Как поднять всё с нуля после перезагрузки Mac

```bash
# 1. Проверить, жива ли Opera с правильным флагом
curl -s http://localhost:9222/json/version
# если 000/connection refused - убить (если висит без флага) и поднять заново:
osascript -e 'tell application "Opera" to quit'  # если запущена без флага
/Applications/Opera.app/Contents/MacOS/Opera --remote-debugging-port=9222 \
  --user-data-dir="$HOME/Library/Application Support/com.operasoftware.Opera"
# логин/VPN обычно восстанавливаются сами из профиля - проверить вкладку p2p.binance.com

# 2. Backend
cd ~/Desktop/Check/backend
source venv/bin/activate
nohup uvicorn main:app --host 0.0.0.0 --port 8088 > /tmp/check_backend.log 2>&1 &

# 3. Проверить
curl -s http://localhost:8088/api/orderbook | python3 -c "import json,sys,time; d=json.load(sys.stdin); print('age:', time.time()-d['timestamp'])"
```

Дашборд: `http://localhost:8088`. После рестарта backend: **auto-reprice выключается**, **active-side override сбрасывается на auto**, **карточка "Рабочая сессия" сбрасывается на дефолтный банк** — все три нужно перепроверить/переустановить вручную, если пользователь их настраивал.

Ключи API — в `backend/.env` (не в git). После правок `pnl_tracker.py` синкать на VPS3: `scp backend/pnl_tracker.py root@187.127.114.34:/root/pnl_tracker.py`.
